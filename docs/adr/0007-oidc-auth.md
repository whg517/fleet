# ADR-0007: 认证采用 OIDC

## 状态

Accepted

## 背景

平台需要统一身份认证，对接企业现有 SSO 系统。
运维、开发、架构师都使用企业账号登录。

## 决策

采用 OIDC (OpenID Connect) 协议对接企业 SSO。
平台不自建用户系统，使用 OIDC 返回的 sub（subject）作为用户唯一标识。

## 后果

### 正面
- 不自建用户系统，减少维护成本
- 统一身份管理，密码策略由 SSO 负责
- 支持多因素认证（由 SSO 提供）
- OIDC 是行业标准，对接简单

### 负面
- 依赖 SSO 可用性（SSO 挂了平台无法登录）
- 首次对接需要与 SSO 团队协调

### 中性
- 需要在平台本地缓存用户基本信息（name、email、role）

## 备选方案

| 方案 | 拒绝理由 |
|------|---------|
| 自建用户系统 | 不维护密码，不符合企业安全规范 |
| SAML | 协议更复杂，OIDC 是现代标准 |
| LDAP 直接绑定 | 安全风险（明文密码），不如 OIDC |
