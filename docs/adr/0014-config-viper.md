# ADR-0014: Go 配置管理采用 Viper

## 状态

Accepted

## 背景

平台需要管理多环境配置（dev/test/pre/prod），配置来源包括：
- 配置文件（YAML）
- 环境变量（K8s 注入）
- 命令行参数

## 决策

采用 **Viper** 作为配置管理方案。

## 后果

### 正面
- 支持多配置源（文件、环境变量、命令行、远程 KV）
- Go 生态最成熟的配置库，文档丰富
- 支持配置热更新（Watch）
- 与 Cobra（CLI 框架）配合好

### 负面
- 库本身偏重，依赖较多
- API 有历史包袱（全局实例 vs 实例化）

### 中性
- 推荐使用 `viper.New()` 实例化方式，避免全局状态

## 备选方案

| 方案 | 拒绝理由 |
|------|---------|
| koanf | 更轻量但社区小 |
| envconfig | 只支持环境变量，功能不够 |
| 标准库 + 手写 | 维护成本高 |
