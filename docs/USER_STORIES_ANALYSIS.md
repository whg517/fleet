## 解决方案

### G-1: 平台级发布窗口

```
admin 可设置全局发布窗口：
  - 发布冻结期（如节假日）：冻结期内所有 pre/prod 部署被阻止
  - 发布开放期：正常流程
  - 紧急豁免：operator 以上角色可申请紧急部署（需额外审批）
```

### G-2: 审批风险评估

```
审批人在审批页面看到风险评估面板：
  - 版本差异：与当前 prod 版本的 changelog
  - 环境验证状态：该版本在 test/pre 是否验证通过
  - 配置变更：是否伴随配置变更
  - 历史回滚率：该服务近 30 天的回滚率
```

### G-3: 版本漂移检测

```
平台定期（每日）扫描所有服务的版本分布：
  - 检测 dev → prod 版本差距超过 N 个版本的服务
  - 检测 prod 版本比 dev 还新的异常情况
  - 检测结果通过 Webhook 通知 + 站内消息
```

### G-4~G-8: 后续迭代

统计分析、资源分析、权限可视化、运营统计、技术债报告统一归入"平台分析"模块，P2 阶段实现。
